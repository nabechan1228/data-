# CarpAnalytics システムガイド & メンテナンス記録

本ドキュメントは、CarpAnalyticsの技術的な構成、および実施された重要な修正内容を記録したものです。

## 1. システム構成・起動方法

### 通信プロトコル
- **ホストの統一**: ブラウザ（特にMicrosoft Edge）における `localhost` の名前解決トラブルを防ぐため、通信はすべて **`127.0.0.1`** (IPv4) に統一しています。
  - バックエンド: `http://127.0.0.1:8001`
  - フロントエンド: `http://127.0.0.1:5173`
- **Vite 開発プロキシ**: `vite.config.js` で `/api` を `http://127.0.0.1:8001` にプロキシしています。フロントの API ベース URLは開発時既定で空（同一オリジン）のため、CORS 設定の負担を減らしています。

### 起動スクリプト (`start_app.ps1`)
- **役割**: バックエンド(Uvicorn)とフロントエンド(Vite)を同時に立ち上げ、適切なURLでブラウザを開きます。
- **変更点**: `package.json` の設定により、Viteが強制的に `127.0.0.1` で待機するように調整済みです。

### リバースプロキシとクライアント IP（管理 API）
- 管理系 `POST /api/update-data`・`/api/update-stats` は、既定では **実接続元 IP** が `127.0.0.1` / `::1` のときのみ許可されます。
- Nginx 等の背後に Uvicorn を置くと、`request.client.host` はプロキシになり、実ブラウザの IP は `X-Forwarded-For` の左端に載ることがあります。
- **安全な有効化**: `.env` で `TRUST_PROXY_HEADERS=true` にし、**接続元が `TRUSTED_PROXY_HOSTS`（既定: `127.0.0.1,::1`）に含まれるときだけ** `X-Forwarded-For` の左端を「実クライアント IP」として扱います。プロキシを別ホストに置く場合は `TRUSTED_PROXY_HOSTS` にそのプロキシの IP を追加してください。信頼できないネットワークから `X-Forwarded-For` を解釈しないでください。
- 本番では `REQUIRE_ADMIN_TOKEN=true` と強い `SCRAPE_SECRET_TOKEN` を併用し、管理操作をトークン必須にすることを推奨します。

## 2. スクレイピング仕様 (`stats_scraper.py`)

### 文字化け対策
- **エンコーディング自動判定**: NPB公式サイトの一部ページが `Shift_JIS (cp932)` で提供されているため、`ISO-8859-1` と誤判定されるケースを検知し、優先的に `cp932` でデコードするロジックを搭載しています。
- **正規化**: 選手名に含まれる外字（`*`や`#`）および空白を自動的に除去し、データベース内でのマッチング精度を高めています。

### 守備成績の抽出
- **動的パース**: NPBのHTML構造（`<h5>`タグでポジションが区切られる形式）に対応。
- **複数ポジション対応**: 坂倉選手などのように、1シーズンで複数の守備位置に就く選手のデータを集計・統合する機能を備えています。

## 3. フロントエンドの最適化

### パフォーマンス向上 (`SeasonRankings.jsx`)
- **O(N) 検索**: 1000名を超える選手データと成績データの照合において、以前の O(N*M) 検索（find）から、`Map` を利用した O(N) 検索へ変更しました。これにより、ランキング表示時のブラウザフリーズを解消しています。

### ブラウザ互換性 (Edge対策)
- **自動翻訳の抑制**: Edgeが `P:` (Potential) を「Page」と誤認して翻訳するのを防ぐため、`translate="no"` 属性を適用しています。
- **レイアウトの安定化**: スクロールバーの影響による「微妙な横揺れ・ズレ」を防ぐため、カスタムスクロールバーおよび固定幅（flex-shrink設定）を導入しています。

### チャート描画 (`PotentialMatrix.jsx`)
- **描画制限**: 1000点以上のドットを一度に動かすと重くなるため、アニメーションを無効化し、表示件数をフィルタリング状況に応じて最適化しています。

## 4. トラブルシューティング

### Q. 「このサイトにアクセスできません」と出る
- `start_app.ps1` を一度閉じ、再度実行してください。ポート 8001 または 5173 が別のプロセスで使用されている場合は、スクリプトが自動的にクリーンアップを試みます。

### Q. 選手データや成績が古い・反映されない
- 画面右上の「成績更新」ボタンをクリックしてください。内部で `stats_scraper.py` と `scraper.py` が連続実行され、データベースが最新化されます。

### Q. 画面がカクつく、または「応答なし」になる
- 「全球団」を表示している場合は負荷が高いため、特定のリーグやチームに絞り込んで表示してください。
